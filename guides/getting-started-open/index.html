Get started with your own Pryv.io

This guide is for Pryv.io owners that have finished the installation of a platform. If you simply wish to test it the API, please refer to the guide for the Pryv Lab.

Prerequisites

You have a compiled, hosted version of app-web-auth3 available.

Development

You must implement a page that launches the consent process. (link to auth request). For the development phase, you can use the Token generator app Which allows the user to input the parameters of the auth request.

Edit the service information URL either in its input Service Info URL or the URL's query parameter serviceInfoUrl. It should be:

  • Pryv.io: https://reg.${domain}/service/info, eg.: https://reg.pryv.me/service/info
  • Open Pryv.io: https://${publicUrl}/reg/service/info, eg.: https://open1.pryv.io/reg/service/info

You can start using a "master" token for the permissions, and press "request access" which will:

  1. Fetch the service information
  2. Perform the auth request

After opening the popup, click on "create an account". (You might have to resize the window or scroll down to make it appear)

Enter your credentials, the sign in and accept the permissions request.

The popup should close and a valid pryvApiEndpoint should be displayed on the Token generator app with a form such as this: coqihe98f9203hn2@my-username.mydomain/
