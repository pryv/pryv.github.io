Standard data structure

In this section, we present all the channels and folders we recommend to use in Pryv. Where appropriate, we also present fictional elements to illustrate how and where you could create your own channels and folders.

We use the following conventions to describe the structure:

  • Recommended means that the concerned channel or folder is predefined and known to Pryv. Its id is reserved and can be used to access it. We advise you to use recommended channels and folders as much as possible, unless the specific needs of your app requires a dedicated channel/folder that doesn't correspond to any presented here.
  • Example is used to represent channels and folders that are given here as examples of elements you could create to fit your app's purposes. They are not predefined in Pryv and thus have no fixed id.

Warning: the event structure used in examples needs an update. Please refer to the API reference for up-to-date info in the meantime.

Channels and folders

Recommended channels and folders are identified by their id.

Recommended channel. Default name (EN): "Diary".

To save all your personal thoughts, pictures and others.

Recommended channel. Default name (EN): "Activity".

Where a user can save her various work and leisure activities.

Recommended channel. Default name (EN): "Health".

This channel is dedicated to the user and her close family's health. The goal here is to save all the data related to one's health: weight, medication, doctor visits, etc.

position channel

Recommended channel. Default name (EN): "Position".

This channel is especially suited for all events automatically saved by the user's devices. Imagine an iphone app recording the user's position every 10 minutes. Here is the perfect place to save this data.

namedescription
